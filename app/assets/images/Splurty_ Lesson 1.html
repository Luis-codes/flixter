Set Up Your Project

To set up your project, start by going to one of the terminals within your coding environment.

Remember, in these instructions, whenever you see a gray box with a dollar sign ($) at the beginning, type the command that follows it in a terminal window (following the dollar sign in that window).

1. Change to the directory where your code will be stored. 

$ cd /vagrant/src

This will change your working directory to the folder where your code is.

Common mistake when changing directories

The above command will change your directory to the location of the source code if you are using the vagrant web development environment.

If you are using an environment that is different from vagrant, replace the /vagrant/src with the folder where the code for your projects will live.

2. Type ls to see a list of other projects in that directory: 

$ ls

3. Create a new application that uses postgres. Let's call this application splurty.

$ rails new splurty --database=postgresql

4. Open your newly created splurty web application code in the Sublime Text editor. Open Sublime Text, click File > Open... (or File > Open Folder in Windows), and navigate to Desktop/vagrant/src/splurty.

5. Tell your Rails application how to connect to the database by editing the config/database.yml file. In Sublime Text, open database.yml and change it so it looks exactly like the code below: 

default: &default
  adapter: postgresql
  encoding: unicode
  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
  username: postgres
  password: password
  host: localhost

development:
  <<: *default
  database: splurty_development

test:
  <<: *default
  database: splurty_test

production:
  <<: *default
  database: splurty_production
  # username: splurty
  # password: <%= ENV['SPLURTY_DATABASE_PASSWORD'] %>

6. When you're done, save the file.

Common Mistakes When Editing Database.yml

1. Make sure adapter = postgresql when you open database.yml. If it doesn't, it means you did not create the application using the --database=postgres command. In this case, delete the entire folder, go back to the rails new step, and continue from there.

2. The database.yml file is whitespace sensitive. Make sure indentation and line spacing are consistent.

Create the Initial Database

Change directory into your project

Whenever you want to work with your project, you first need to make sure your terminal window's current directory is the project you have created.

Keep in mind, if you're not inside your project folder, the commands will not work and these commands will fail.

In this case, the current directory should be changed to the splurty folder. This means you should run the cd command to navigate to the folder.

For example, if you're using the vagrant environment, the splurty folder will have the full directory /vagrant/src/splurty. You can navigate into this directory, by running the following command. 

$ cd /vagrant/src/splurty

Create your Initial Database

At the beginning of each new web application, you need to create an initial, empty database so you have a place to store information. For this one time, type this command: 

$ rake db:create

This command outputs that it's creating your development and test environments. When it's done, it will return a dollar sign: 

Created database 'splurty_development'
Created database 'splurty_test'

It's okay if it says these environments exist already. If it gives you an error message, check your database.yml file to double check that it matches the code that was shown earlier in this lesson.

Start the Server

Now that you've configured database.yml and created your initial database, it's time to fire up the server.

Start the server by running the following command: 

$ rails server -b 0.0.0.0 -p 3000

This command will look like it's running endlessly because it won't return a dollar sign. That's a good thing, because you want the server to be constantly running in the background so you can continuously see your Splurty web application in the browser. You will let the server run in this terminal window, and use the other one to start your web application.

In the second terminal window, type the following command to move into the splurty project folder: 

cd /vagrant/src/splurty

Test Your Server

Preview the application you created by running those steps in a web browser. For example, if you're using a vagrant environment, by visiting http://localhost:3030 in your browser, you will be able to preview the application. You should see something like this:
